Testing basic requests

Requesting existing ex2's files, and checking for:
- Status 200
- Content-Type header exists and corresponds to the file type.
- Content-Length header exists and corresponds to the file size.
- Date header exists

Testing Non-existing file requests

Requesting a Non-existing file, checking for:
- Status 404 (Not Found)

Testing non-HTTP requests

Sending the server some bytes, i.e. non-HTTP-request, and checking for:
- Status 500 (Internal Server Error).

Testing directory traversal is disallowed

Requesting a file outside the root directory (i.e. with '../'), and checking for:
- Status 403 (Forbidden).

Testing JSON middleware + PUT requests

- Testing the json-middleware parses the request body correctly (in case it's JSON).
- Testing the response.json() function works + sets type to 'application/json'.
- Testing PUT requests (using app.put()).

Testing urlencoded middleware + POST requests

- Testing the urlencoded-middleware parses the request body correctly (in case it's type is 'application/x-www-form-urlencoded').
- Testing POST requests (using app.post()).

Testing bodyParser middleware + DELETE requests

- Testing the bodyParser-middleware parses the request body correctly (JSON or urlencoded).
- Testing DELETE requests (using app.delete()).

Testing Illegal methods are not accepted

Testing that non-GET/POST/PUT/DELETE requests return error 405 (method not allowed).
Notice non-HTTP methods will be responded with error 500 (non-HTTP-request), whereas HTTP-methods that are not
supported (such as OPTIONS, TRACE..) will be responded with a 405.

Testing request parsing (params, query, cookies)

- Testing that req.params is populated corretcly according to the request.
- Testing that req.query is populated corretcly according to the query (in path).
- Testing that req.cookies is populated corretcly according to the 'Cookie' header.
- Testing that req.param() returns the corresponding parameter.
- Testing that req.is() returns the correct answer according to the content-type.
- - - Testing 3 formats: 'text/html', 'html', 'text/*' - all should return true for 'text/html'.
- Testing that req.host is populated correctly (and that an absolute url is parsed correctly to host+path).

Testing setting cookies

- Testing that resp.cookie() creates a corresponding Set-Cookie header.
- Testing that multiple 'Set-Cookie' headers are allowed in the same response.
- Testing that the cookie's default path is '/'.
- Using both 'Expires' and 'MaxAge'.

Testing only the first matched-route send() occurs

Testing only the first matched-route can send() to client, by defining 3 matching routes
and checking that only the first one sends a response.

Testing all matching routes are taken in case none of them used send()/json()

Testing the request has taken all (3) matching routes as long no send()/json() was used.
